# Visual Design Critique

Review layouts, typography, accessibility, color, hierarchy, responsive behavior, and product polish.

## Activation

Use this skill when the user asks about: design review, UI critique, accessibility, layout, visual polish.

## Workflow

1. Evaluate clarity, hierarchy, contrast, spacing, alignment, affordance, and responsive stability.
2. Check accessibility basics including readable text, keyboard paths, focus, and color contrast.
3. Prioritize changes by user impact and implementation effort.
4. Offer concrete revisions rather than vague taste notes.

## Expected Outputs

- design findings
- revision checklist
- accessibility notes

## Safety And Quality Boundaries

- Do not critique aesthetics without tying recommendations to usability or brand goals.

- Label assumptions and uncertainty.
- Protect private data and secrets.
- Prefer current authoritative references for changing, regulated, or high-stakes topics.
- When the user writes in Portuguese, produce the artifact in Portuguese unless they request another language.
