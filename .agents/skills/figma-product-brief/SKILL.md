# Figma Product Brief

Prepare Figma-ready product briefs, user flows, component inventories, and design handoff notes.

## Activation

Use this skill when the user asks about: Figma, wireframe, UX, UI, product design, prototype.

## Workflow

1. Define user, job-to-be-done, platform, states, constraints, and success metric.
2. Map primary flow, secondary flows, empty/error/loading states, and accessibility requirements.
3. List components, content hierarchy, responsive rules, and handoff questions.
4. Create prompts or checklists a designer can use in Figma.

## Expected Outputs

- product brief
- flow map
- component inventory
- handoff checklist

## Safety And Quality Boundaries

- Do not promise pixel-perfect output without actual design review or Figma access.

- Label assumptions and uncertainty.
- Protect private data and secrets.
- Prefer current authoritative references for changing, regulated, or high-stakes topics.
- When the user writes in Portuguese, produce the artifact in Portuguese unless they request another language.
