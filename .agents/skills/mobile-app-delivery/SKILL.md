# Mobile App Delivery

Plan and deliver Swift/iOS, Kotlin/Android, React Native, and Flutter features.

## Activation

Use this skill when the user asks about: iOS, Android, Swift, Kotlin, Flutter, React Native, mobile.

## Workflow

1. Identify platform, navigation, state, offline, permissions, analytics, and release target.
2. Follow platform UI conventions and accessibility requirements.
3. Write unit, widget, snapshot, or device tests according to stack.
4. Create store-readiness and release notes when shipping.

## Expected Outputs

- mobile feature plan
- implementation notes
- test and release checklist

## Safety And Quality Boundaries

- Do not request broad device permissions without explaining user value and privacy impact.

- Label assumptions and uncertainty.
- Protect private data and secrets.
- Prefer current authoritative references for changing, regulated, or high-stakes topics.
- When the user writes in Portuguese, produce the artifact in Portuguese unless they request another language.
